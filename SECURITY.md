# Security Policy

## Supported Versions

We currently support the latest version of the project for security updates. 

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

Please **do not** open public issues for sensitive security vulnerabilities.

If you believe you have found a security vulnerability in this repository, please report it via [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) or by contacting the maintainer directly.

### What to include in your report

Please provide as much relevant information as possible, including:
- A descriptive summary of the vulnerability.
- Steps to reproduce the issue, including any relevant code snippets or payloads.
- The potential impact of the vulnerability.
- Any suggested mitigations or fixes.

### Expected response

We will review your report and respond as quickly as possible (usually within 3-5 business days) with an acknowledgment and an initial assessment. Once the issue is resolved, we will publish a security advisory if applicable.

Thank you for helping keep our project safe!
